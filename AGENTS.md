# Codex 文档格式执行规范

本文件用于约束 Codex 在本仓库或引用本 Skill 的项目中处理文档类任务时的行为。

## 1. 总体原则

- 处理文档时，优先遵守 `SKILL.md` 中的文档格式与 Markdown 转 DOCX 规范。
- 不得把 Markdown 源文档原样塞入 DOCX。
- 不得在 Word 正文中残留 Markdown 标记符号。
- 不得因为格式清洗改变原文事实、技术路线、章节含义和项目边界。
- 格式修改和内容改写必须分开处理。
- 长文档修改时，不要整篇打印到聊天界面，应输出修改摘要、问题清单和生成文件路径。

## 2. Markdown 转 DOCX 必做流程

当任务涉及 Markdown 转 Word/DOCX 时，必须按以下顺序执行：

1. 读取 Markdown 源文档。
2. 识别标题、正文、列表、表格、代码块、图片、链接。
3. 清理 Markdown 残留符号。
4. 将标题映射为 Word 标题样式。
5. 将 Markdown 表格转换为真实 Word 表格。
6. 将代码块转换为等宽字体段落，不保留反引号围栏。
7. 将 `---`、`***`、`___` 等分割线删除或转换为分页/段落分隔。
8. 统一正文缩进、行距、段前段后和列表编号。
9. 设置页边距、页码、必要目录、图表编号。
10. 生成 DOCX 后执行残留检查和交付前验收。
11. 输出修改摘要和交付文件路径。

## 3. DOCX 残留检查清单

生成 DOCX 后必须检查正文是否仍包含以下内容：

- `#`、`##`、`###` 标题残留。
- `**`、`__`、`~~` 等强调标记。
- 单独一行 `---`、`***`、`___`。
- 三个反引号代码围栏。
- Markdown 表格管道符 `|`。
- Markdown 链接格式 `[标题](链接)`。
- Markdown 图片格式 `![说明](路径)`。
- 多余转义符 `\*`、`\#`、`\_`。

只要存在上述残留，就不能视为转换完成。

## 4. Word 排版硬性要求

- 页面：A4，纵向。
- 页边距：上 2.5 cm、下 2.5 cm、左 2.8 cm、右 2.8 cm。
- 文档标题：黑体，二号或小二号，居中，加粗。
- 一级标题：黑体，三号，居左，加粗。
- 二级标题：黑体，小三号，居左，加粗。
- 三级标题：黑体，四号，居左，加粗。
- 正文：宋体或仿宋，四号或小四号，两端对齐，首行缩进 2 字符。
- 正文行距：1.5 倍行距或固定值 28 磅，全文统一。
- 正文段前：0 磅。
- 正文段后：0 磅或 6 磅，全文统一。
- 表格文字：宋体或仿宋，五号。
- 代码块：Consolas 或 Courier New，五号。
- 正文字体颜色统一为黑色。

## 5. 编号与列表要求

- 标题编号使用：1、1.1、1.1.1。
- 正文第一层列举使用：（1）（2）（3）。
- 正文第二层列举使用：1）2）3）。
- 正文第三层列举使用：a）b）c）。
- 同一层级不得混用短横线、圆点、①、1）、一是等格式。
- Markdown 无序列表转 DOCX 时，不保留 `-`、`*`、`+` 原始符号。

## 6. 目录、页码、图表要求

- 正式 DOCX 建议生成自动目录。
- 目录只包含一级至三级标题。
- 不得手工输入目录页码。
- 正文首页页码从 1 开始，页码位于页脚居中。
- 表题置于表格上方，例如“表 1 模块功能清单”。
- 图题置于图片下方，例如“图 1 系统总体架构”。
- 正文引用图表时使用“如图 1 所示”“见表 2”，不使用“上图”“下表”。

## 7. 内容审查要求

处理技术方案、设计文档、模块说明时，应检查：

- 是否说明模块定位、输入数据、处理流程、输出结果、异常处理、验收标准。
- 是否存在空泛表述、营销语言、AI 化套话。
- 是否将未实现能力写成已完成能力。
- 是否编造未确认的技术指标、性能指标、模型能力或数据规模。
- 是否明确开源项目的使用方式：直接复用、接口调用、数据源接入或架构参考。

## 8. 交付前验收清单

交付 DOCX 前必须检查：

1. 是否无 Markdown 残留。
2. 标题编号是否连续。
3. 正文是否两端对齐、首行缩进 2 字符。
4. 正文行距、段前段后是否统一。
5. 正文列表编号是否统一。
6. 目录页码是否更新。
7. 页码是否正常。
8. 表格是否为真实 Word 表格，是否超出页边距。
9. 长表格跨页时是否重复表头。
10. 图题、表题编号是否连续。
11. 是否存在大段空白页、孤立标题、异常分页。
12. 是否存在明显 AI 化套话和空泛描述。

## 9. 输出要求

- 简单修改可以直接说明修改点。
- 长文档修改必须生成文件，不要在对话中整篇输出。
- 审查类任务优先输出编号清单。
- 转换类任务必须说明是否完成 Markdown 残留清理和 DOCX 交付前验收。
