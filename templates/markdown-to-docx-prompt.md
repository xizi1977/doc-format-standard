# Markdown 转 DOCX 任务提示词模板

将下面提示词复制给 Codex 或其他 AI 编程工具使用。

```text
请按照项目根目录 AGENTS.md 和 doc-format-standard 规范，将指定 Markdown 文档转换为正式 DOCX 文档。

输入文件：{{input_md_path}}
输出文件：{{output_docx_path}}

必须执行以下要求：

1. 读取 Markdown 源文档，识别标题、正文、列表、表格、代码块、图片和链接。
2. 清理 Markdown 残留符号，DOCX 正文中不得出现：
   - #、##、###
   - **、__、~~
   - ---、***、___
   - 三个反引号代码围栏
   - Markdown 表格管道符 |
   - [标题](链接)
   - ![说明](路径)
3. 将 Markdown 标题映射为 Word 标题样式。
4. 将 Markdown 表格转换为真实 Word 表格，不得保留管道符。
5. 将代码块转换为等宽字体段落，不保留反引号围栏。
6. 删除或转换 Markdown 分割线，不得在 DOCX 中保留单独一行 ---。
7. 设置 Word 页面格式：A4、纵向、页边距上 2.5 cm、下 2.5 cm、左 2.8 cm、右 2.8 cm。
8. 设置字体字号：
   - 文档标题：黑体，二号或小二号，居中，加粗；
   - 一级标题：黑体，三号，居左，加粗；
   - 二级标题：黑体，小三号，居左，加粗；
   - 三级标题：黑体，四号，居左，加粗；
   - 正文：宋体或仿宋，四号或小四号；
   - 表格文字：宋体或仿宋，五号；
   - 代码块：Consolas 或 Courier New，五号。
9. 设置正文段落：两端对齐、首行缩进 2 字符、行距统一、段前段后统一。
10. 统一编号格式：
   - 标题编号：1、1.1、1.1.1；
   - 正文第一层列举：（1）（2）（3）；
   - 正文第二层列举：1）2）3）；
   - 正文第三层列举：a）b）c）。
11. 如文档包含表格和图片，补充表题和图题：
   - 表题位于表格上方，例如“表 1 模块功能清单”；
   - 图题位于图片下方，例如“图 1 系统总体架构”。
12. 正式文档需要目录时，生成自动目录，目录只包含一级至三级标题。
13. 设置页码，正文页码位于页脚居中。
14. 生成 DOCX 后执行交付前检查，确认无 Markdown 残留、编号连续、缩进统一、表格正常、图表编号连续。
15. 不要改变原文事实、技术路线、章节含义和项目边界。
16. 不要在聊天界面打印全文，只输出：
   - 已完成的修改摘要；
   - 发现并处理的问题；
   - 生成的 DOCX 文件路径；
   - 是否通过交付前检查。
```
