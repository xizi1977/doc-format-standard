# doc-format-standard

一套面向 AI 生产文档的格式与行文规范 Skill，适用于 [Manus](https://manus.im) 平台。

## 简介

本 Skill 将"文档格式与行文规范要求"结构化为 Manus 可直接调用的技能模块，用于约束 AI 生成文档的排版、行文风格和内容质量，避免 AI 产出的文档出现格式混乱、行文空泛、AI 味过重等问题。

## 适用场景

- 生成技术方案文档
- 编写功能模块说明
- 撰写项目交付物（Markdown / Word / PDF）
- 对已有文档进行格式规范化审查

## 规范涵盖内容

| 章节 | 说明 |
|------|------|
| 页面与版式 | A4 纸张、字体颜色、段落间距等基础版式要求 |
| 标题与编号 | 统一层级编号格式，禁止混用多套编号体系 |
| 正文写作 | 面向开发人员，突出输入/输出/边界/验收 |
| 去 AI 化行文 | 避免模板套话、宏大表述、营销语言 |
| 表格与代码块 | 字段命名统一，代码块需前后说明用途 |
| 图示 | 黑白线框图风格，术语与正文保持一致 |
| 交付要求 | Markdown + Word + PDF 三版本同步交付 |

## 安装方式

将 `doc-format-standard/` 目录整体复制到 Manus 的 Skills 目录下：

```bash
cp -r doc-format-standard/ ~/skills/
```
