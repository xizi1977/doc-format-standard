# DOCX 交付前检查清单

本清单用于 Markdown 转 DOCX、AI 生成 Word 文档、技术方案排版和正式交付前审查。

## 1. Markdown 结构识别检查

- [ ] 已识别代码围栏范围，并将代码块标记为保护区。
- [ ] 代码块内的 Markdown 示例未被误解析为正文标题、正文表格或正文链接。
- [ ] 已确认标题映射关系：`#` 文档标题、`##` 一级标题、`###` 二级标题、`####` 三级标题。
- [ ] 如果标题已有手工编号，未再叠加 Word 自动编号。

## 2. Markdown 残留检查

交付前确认 DOCX 正文区不存在以下残留：

- [ ] `#`、`##`、`###`、`####` 标题符号。
- [ ] `**`、`__`、`~~` 等强调标记。
- [ ] 单独一行 `---`、`***`、`___`。
- [ ] 三个反引号代码围栏。
- [ ] Markdown 表格管道符 `|`。
- [ ] Markdown 链接格式 `[标题](链接)`。
- [ ] Markdown 图片格式 `![说明](路径)`。
- [ ] HTML 换行标签 `<br>`、`<br/>`、`<br />`。
- [ ] 多余转义符 `\*`、`\#`、`\_`。

说明：代码块区可以保留示例内容中的 `#`、`|` 等字符，但必须使用代码块样式呈现，且不得保留反引号围栏。

## 3. 页面与字体检查

- [ ] 页面为 A4，纵向。
- [ ] 页边距为上 2.5 cm、下 2.5 cm、左 2.8 cm、右 2.8 cm。
- [ ] 正文字体颜色统一为黑色。
- [ ] 文档标题、一级标题、二级标题、三级标题字号符合规范。
- [ ] 正文使用宋体或仿宋，四号或小四号。
- [ ] 表格文字使用五号。
- [ ] 代码块使用 Consolas 或 Courier New，五号。

## 4. 段落与缩进检查

- [ ] 正文两端对齐。
- [ ] 正文首行缩进 2 字符。
- [ ] 正文行距全文统一。
- [ ] 正文段前段后全文统一。
- [ ] 标题、表格、图题、表题、代码块未套用正文首行缩进。
- [ ] 未使用连续空格、Tab 或多个空行模拟缩进和间距。
- [ ] 不存在大段空白页、孤立标题或异常分页。

## 5. 编号与列表检查

- [ ] 标题编号使用 1、1.1、1.1.1。
- [ ] 未出现“1 1 项目定位”这类重复编号。
- [ ] 正文第一层列举使用（1）（2）（3）。
- [ ] 正文第二层列举使用 1）2）3）。
- [ ] 正文第三层列举使用 a）b）c）。
- [ ] 同一层级未混用短横线、圆点、①、1）、一是等格式。
- [ ] Markdown 无序列表未残留 `-`、`*`、`+`。

## 6. 加粗标签与字段说明检查

- [ ] `**模块定位**：` 已转换为“模块定位：”加粗标签。
- [ ] `**输入数据**：` 已转换为“输入数据：”加粗标签。
- [ ] `**处理流程**：` 已转换为“处理流程：”加粗标签或小标题。
- [ ] `**输出结果**：` 已转换为“输出结果：”加粗标签。
- [ ] `**异常处理**：` 已转换为“异常处理：”加粗标签。
- [ ] `**验收标准**：` 已转换为“验收标准：”加粗标签。
- [ ] 未在 DOCX 中残留 `**`。

## 7. 表格检查

- [ ] Markdown 表格已转换为真实 Word 表格。
- [ ] 表题位于表格上方。
- [ ] `表1` 已规范为“表 1”。
- [ ] 表头加粗。
- [ ] 表格未超出页边距。
- [ ] 长表格跨页时重复表头。
- [ ] 表格列名和字段命名保持统一。
- [ ] 表格单元格内的 `<br>` 已转换为单元格内换行。
- [ ] 未将表格转换为截图。

## 8. 代码块检查

- [ ] 代码块未保留反引号围栏。
- [ ] JSON、SQL、HTTP、Markdown、text 示例保留原有语义。
- [ ] 代码块未被误套正文首行缩进。
- [ ] 超过 30 行的长代码块或目录树未造成严重分页破碎。

## 9. 图示检查

- [ ] 图示为白底黑字或正式技术图风格。
- [ ] 图题位于图片下方。
- [ ] `图1` 已规范为“图 1”。
- [ ] 图编号全文连续。
- [ ] 正文引用图表时使用“如图 1 所示”“见表 2”。
- [ ] 未使用“上图”“下表”等相对引用。

## 10. 目录、页码与页眉页脚检查

- [ ] 正式文档已生成自动目录。
- [ ] 目录只包含一级至三级标题。
- [ ] 目录未包含代码块中的 Markdown 标题示例。
- [ ] 目录页码已更新。
- [ ] 正文页码正常，页码位于页脚居中。
- [ ] 未添加复杂水印、彩色页脚或装饰线。

## 11. 内容质量检查

- [ ] 不存在明显 AI 化套话。
- [ ] 不存在空泛表达和营销式语言。
- [ ] 未将未实现能力写成已完成能力。
- [ ] 未编造未确认的技术指标、性能指标、模型能力或数据规模。
- [ ] 开源项目使用方式已明确为直接复用、接口调用、数据源接入或架构参考。
- [ ] 模块描述包含模块定位、输入数据、处理流程、输出结果、异常处理、验收标准。

## 12. 交付一致性检查

- [ ] Markdown、DOCX、PDF 三个版本的标题一致。
- [ ] Markdown、DOCX、PDF 三个版本的章节编号一致。
- [ ] Markdown、DOCX、PDF 三个版本的图表编号一致。
- [ ] 文件名、版本号、日期符合交付要求。
